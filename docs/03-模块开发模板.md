# 模块开发模板

> 如何在 10 分钟内创建一个新模块。

## 概述

每个模块都是一个独立的 Python 包，放在 `src/modules/<module_name>/` 中。

模块必须：
1. 继承 `BaseModule`
2. 定义 `info`（ModuleInfo 元数据）
3. 实现 `check()`（只读诊断）
4. 可选实现 `execute()`（操作指令）

## 10 步创建新模块

### 第 1 步：创建目录

```bash
mkdir -p src/modules/<module_name>
```

命名规则：
- 小写字母 + 下划线
- 必须以字母开头
- 2-50 个字符
- 例如：`health`, `clean_downloads`, `github_mgr`

### 第 2 步：创建文件

```
src/modules/<module_name>/
├── __init__.py          # 空文件
├── module.py            # 模块主类（必须）
├── <helper>.py          # 核心逻辑（可选，复杂模块才需要）
└── __main__.py          # 独立运行入口（推荐）
```

最少只需 `__init__.py` + `module.py`。

### 第 3 步：定义模块信息

```python
# module.py
from src.core.module_base import BaseModule, ModuleResult, ModuleInfo

class MyModule(BaseModule):
    info = ModuleInfo(
        name="my_module",           # 唯一标识（必须）
        version="0.1.0",            # 语义版本（必须）
        description="我的模块描述",   # 一行描述（必须）
        category="tools",           # 分组：maintenance / tools / automation / general
        icon="🔧",                  # 图标（可选）
    )
```

### 第 4 步：实现 check() — 必需的只读诊断

```python
    def check(self) -> ModuleResult:
        """
        检查模块是否可用。

        check() 规则：
        - 不能修改任何文件或状态
        - 可以随时安全运行
        - 应该比较快（< 1 秒）
        - 必须返回 ModuleResult
        """
        # 1. 收集数据
        data = {"status": "ok", "version": "1.0"}

        # 2. 检查问题
        errors = []
        warnings = []

        if some_problem:
            errors.append("出错了")

        if minor_issue:
            warnings.append("有个小问题")

        # 3. 返回结果
        if errors:
            return ModuleResult.fail(self.info.name, errors, warnings)

        return ModuleResult.ok(self.info.name, data, warnings)
```

### 第 5 步：可选实现 execute() — 操作指令

```python
    def execute(self, **kwargs) -> ModuleResult:
        """
        执行操作。可能有副作用（创建文件、调用 API 等）。

        **kwargs 由 CLI 传入：yqa run my_module arg1=value1
        """
        # 1. 解析参数
        target = kwargs.get("target", "default")

        # 2. 执行操作
        try:
            result = do_something(target)
        except Exception as e:
            return ModuleResult.fail(self.info.name, [str(e)])

        # 3. 返回结果
        return ModuleResult.ok(self.info.name, {"result": result})
```

如果不重写 `execute()`，默认返回「不支持 execute()」的错误。

### 第 6 步：创建 __main__.py（可选但推荐）

```python
# __main__.py
"""允许模块独立运行: python -m src.modules.my_module"""

from src.modules.my_module.module import MyModule

if __name__ == "__main__":
    mod = MyModule()
    result = mod.check()
    print(f"Check: {result.is_ok}")
    print(result)
```

### 第 7 步：测试 CLI

```bash
# 列出模块
yqa list

# 运行诊断
yqa check my_module

# 执行操作
yqa run my_module target=hello

# 查看状态
yqa status
```

### 第 8 步：写测试

```python
# tests/modules/test_my_module.py
from src.modules.my_module.module import MyModule

def test_check():
    mod = MyModule()
    result = mod.check()
    assert result.is_ok

def test_execute():
    mod = MyModule()
    result = mod.execute(target="test")
    assert result.success
```

运行测试：
```bash
pytest tests/modules/test_my_module.py -v
```

### 第 9 步：写文档

在模块目录中添加 `README.md`（可选但推荐）：
```markdown
# My Module
功能描述...
用法: yqa run my_module
```

### 第 10 步：注册到项目

**不需要修改任何核心代码。** ModuleManager 会自动发现你的模块。

只要目录放在 `src/modules/` 下，`module.py` 包含 `BaseModule` 子类，就会被自动发现。

## 完整模板

```python
"""
My Module — 功能描述.

用法:
    python -m src.modules.my_module
    yqa check my_module
    yqa run my_module
"""

from src.core.module_base import BaseModule, ModuleResult, ModuleInfo


class MyModule(BaseModule):
    """模块类的 docstring 描述它的职责。"""

    # ── 元信息（必须）─────────────────────
    info = ModuleInfo(
        name="my_module",
        version="0.1.0",
        description="一句话描述",
        category="general",
        icon="📦",
    )

    # ── 配置（可选）───────────────────────
    def __init__(self, config=None, event_bus=None):
        super().__init__(config, event_bus)
        # 初始化模块自己的资源
        self.ready = True

    # ── 诊断（必须）───────────────────────
    def check(self) -> ModuleResult:
        errors = []
        warnings = []

        try:
            # 你的检查逻辑
            data = {"status": "healthy"}
        except Exception as e:
            errors.append(str(e))
            data = None

        if errors:
            return ModuleResult.fail(self.info.name, errors, warnings)
        return ModuleResult.ok(self.info.name, data, warnings)

    # ── 操作（可选）───────────────────────
    def execute(self, **kwargs) -> ModuleResult:
        target = kwargs.get("target", "default")

        try:
            # 你的执行逻辑
            result = f"Processed: {target}"
        except Exception as e:
            return ModuleResult.fail(self.info.name, [str(e)])

        return ModuleResult.ok(self.info.name, {"result": result})

    # ── 状态（可选）───────────────────────
    def get_status(self) -> dict:
        status = super().get_status()
        status["ready"] = self.ready
        return status


# ── 独立运行入口 ──────────────────────────
if __name__ == "__main__":
    import sys

    mod = MyModule()

    if len(sys.argv) > 1 and sys.argv[1] == "run":
        result = mod.execute()
    else:
        result = mod.check()

    print(f"Module: {result.module_name}")
    print(f"Success: {result.success}")
    if result.errors:
        print(f"Errors: {result.errors}")
    if result.warnings:
        print(f"Warnings: {result.warnings}")
```

## 常见问题

### Q: 我的模块需要访问外部 API，怎么做？

A: 使用 `services/` 层的服务。不要直接 import requests 或用 subprocess 调用。

```python
# Good: 使用 Service 层
from src.services.network_service import NetworkService
ns = NetworkService(proxy_config=self.config.get("proxy"))
response = ns.get("https://api.example.com")

# Bad: 直接调用
import requests
response = requests.get("https://api.example.com")
```

### Q: 我的模块需要和其他模块通信，怎么做？

A: 使用 EventBus。不要直接 import 其他模块。

```python
# Good: 发布事件
self.event_bus.publish("my_module:something_happened", {"data": ...})

# Bad: 直接调用
from src.modules.other_module.module import OtherModule
OtherModule().do_something()
```

### Q: 我的模块需要保存数据，怎么做？

A: 使用 FileSystemService。

```python
from src.services.filesystem_service import FileSystemService
fs = FileSystemService()
fs.write_json("C:/Users/L1372/.yqa/mymodule/data.json", {"key": "value"})
```

### Q: check() 应该在什么情况下返回 fail？

A: 当模块的核心功能无法使用时。例如：
- GitHub 模块：Token 未配置 → fail
- Health 模块：关键工具未安装 → fail
- Backup 模块：备份目录不存在 → fail

小问题用 warnings，不影响 `is_ok`。
